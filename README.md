# Threat Analysis Case Studies 

Real-world cybersecurity incident analysis from the SOC analyst perspective.

Each case study reconstructs the alerts as they appeared to the analyst in real time, before the incident was publicly known. The goal is to document the triage reasoning applied to each alert, identify what was missed, and extract lessons for modern SOC operations.

This is not hindsight analysis. It is an exercise in thinking like an analyst under pressure, with incomplete information, before knowing the outcome.

## Case Studies 🇺🇸

| Incident | Year | Threat Actor | Key Techniques |
|----------|------|-------------|----------------|
| [SolarWinds Sunburst](./threat-analysis-solarwinds-sunburst-2020.md) | 2020 | APT29 | Supply Chain, DGA, LotL, Low and Slow Exfiltration |
| [Uber Security Breach](./threat-analysis-uber-breach-2022.md) | 2022 | Lapsus$ | MFA Fatigue, Social Engineering, Internal Spearphishing |
| [Colonial Pipeline Ransomware](./threat-analysis-colonial-pipeline-2021.md) | 2021 | DarkSide | Dormant Credentials, No MFA, Double Extortion, IT/OT Segmentation |
## Methodology

Each case study follows this structure: 
1. Incident background
2. Real alerts as they appeared to the SOC
3. My triage reasoning for each alert
4. What the real SOC missed and why
5. Key lessons extracted

# Estudos de Caso — Análise de Ameaças

Análise de incidentes reais de cibersegurança sob a perspectiva do analista SOC.

Cada estudo reconstrói os alertas exatamente como apareceram para o analista no momento do incidente, antes de o ataque ser publicamente conhecido. O objetivo é documentar o raciocínio de triagem aplicado a cada alerta, identificar o que foi ignorado e extrair lições para operações SOC modernas.

Não é análise retrospectiva com o gabarito em mãos. É um exercício de pensar como analista sob pressão, com informação incompleta, sem saber o desfecho.
